## Реализация распараллеливания

При запуске программы на GPU создается сетка потоков. Каждый поток отвечает за обработку одного пикселя выходного изображения. 

Потоки объединяются в блоки размером 16×16. Количество блоков вычисляется исходя из размеров изображения: ширина и высота делятся на 16 с округлением вверх. 

Каждый блок выполняется на отдельном вычислительном блоке GPU. Потоки внутри блока работают параллельно и не требуют синхронизации друг с другом, так как не обмениваются данными.

## Использование текстурной памяти

Согласно требованию лабораторной работы, доступ к пикселям исходного изображения выполняется через текстурную память. 

Для этого в программе создается специальный объект `cudaTextureObject_t`, который привязывается к массиву с изображением. Внутри ядра чтение пикселей происходит через функцию `tex2D`. 

Текстурная память выгодна тем, что имеет аппаратное кэширование и оптимизирована для доступа к соседним элементам, что важно при чтении окна 3×3.
